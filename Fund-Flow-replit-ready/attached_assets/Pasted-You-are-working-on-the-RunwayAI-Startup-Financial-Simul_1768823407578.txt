You are working on the “RunwayAI – Startup Financial Simulator” project.

## Goal
Enhance the dashboard for founders by adding key unit‑economic and financial health metrics, and remove the existing “Benchmark Comparisons” section entirely.

## Requirements

1. **Remove Benchmark Section**
   - Identify and remove the UI component(s) that display the “Benchmark Comparisons” section (including its individual charts for Revenue Growth MoM, Gross Margin, Burn Multiple and Runway P50). Ensure the dashboard layout remains cohesive after removal.

2. **Add New Metrics**
   Implement new cards or panels similar in style to the existing top‑row metrics for the following items:
   - **Customer Acquisition Cost (CAC)**
   - **Lifetime Value (LTV)**
   - **LTV:CAC Ratio**
   - **Net Dollar Retention (NDR)**
   - **Churn Rate** (customer and revenue)
   - **Payback Period**
   - **Monthly Recurring Revenue (MRR) / Annual Recurring Revenue (ARR)**
   - **Average Revenue per User/Account (ARPU/ARPA)**
   - **Monthly Burn Breakdown** – include a simple stacked bar chart or breakdown showing major expense categories (e.g. payroll, marketing, R&D, G&A).
   - **Scenario‑Based Runway** – add an interactive component to toggle between current burn, aggressive growth, and cost‑cut scenarios, updating the runway figure accordingly.
   - **Cash Flow Forecast** – provide a mini‑chart forecasting cash inflows and outflows over the next 12 months.
   - **Headcount & Hiring Plans** – show current headcount and planned hires, and compute revenue per employee.
   - **Segmented Revenue Growth** – allow breakdown of revenue by product line or customer segment if data is available.

3. **Alerts & Warnings**
   - Reuse the existing “Warnings & Insights” section to include alerts for rising churn, increasing CAC, or deteriorating LTV:CAC ratio. Each alert should display when a metric crosses a defined threshold (e.g. churn > 5%, CAC up 20% month‑over‑month, LTV:CAC < 3).

4. **Data Model**
   - Define or extend the underlying data model (e.g. JSON or database schema) to support the new metrics. Where necessary, use mock or placeholder values for demonstration.
   - Ensure calculations (e.g. LTV:CAC, payback period) are derived from available data fields.

5. **UI/UX Consistency**
   - Match the styling, typography and color scheme of the existing dashboard for all new components.
   - Organize the layout so the most critical metrics (cash runway, burn, revenue) remain prominent, while the new metrics are grouped logically (e.g. unit economics, revenue metrics, headcount).

6. **Documentation**
   - Include comments in the code explaining where new metrics are calculated and how they should be updated when real data is integrated.

After implementing these changes, the dashboard should display actionable metrics for founders without the benchmarking section, while still maintaining a clean, modern look.
