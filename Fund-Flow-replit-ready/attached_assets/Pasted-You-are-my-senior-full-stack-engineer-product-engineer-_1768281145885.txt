B) CLASSIFICATION + NORMALIZATION ENGINE
========================================

Implement a deterministic scoring-based classifier for row labels + values.

B1) Row Label Parsing + Hierarchy

* Parse indentation/hierarchy from Excel row labels:

  * If label begins with spaces, bullets, or obvious indentation markers → treat as child under last non-indented header.
  * Maintain parent->children mapping.

B2) Sign Convention Detection (File-level)
From parsed rows across periods:

* Identify candidate expense-like rows via keywords (see B3 expense keywords).
* If >60% of candidate expense-like row values are negative AND revenue-like rows are mostly positive → sign_convention="accounting"
* Else if most values are positive → "all_positive"
* Else → "mixed"
  Store in ImportSession.detected_sign_convention.

B3) Classification Scoring
For each row label (case-insensitive, normalized whitespace):
Compute scores:

Revenue keywords (boost):

* "revenue", "revenues", "sales", "gmv", "bookings", "collections", "income" (careful: "operating income" is derived)
  Expense keywords (boost):
* "expense", "expenses", "cost", "cogs", "opex", "payroll", "salary", "wages", "marketing", "ads", "rent", "hosting", "cloud",
  "logistics", "packaging", "commission", "payout", "contractor", "support", "payment processing", "fulfillment"
  Derived/KPI keywords (boost):
* "gross profit", "operating income", "ebit", "ebitda", "margin", "rule of", "magic number", "burn multiple", "ltv", "cac", "payback"
  Header/Section rows (detect exact matches OR common group headings):
* exact label in ["revenue", "cogs", "sales and marketing", "other opex", "gross profit", "operating income"]
  Also treat rows with children as "header" (unless clearly leaf).

Tie-breakers using values:

* If values are consistently negative across months → strongly favor "expense"
* If values are consistently positive and label is revenue-like → favor "revenue"
* If derived keyword present → "derived" no matter sign

Final decision:

1. If derived_score highest and above threshold → classification="derived"
2. Else if header detected or has children → classification="header"
3. Else if expense_score > revenue_score and above threshold → classification="expense"
4. Else if revenue_score above threshold → classification="revenue"
5. Else "other"

B4) Expense Bucket Mapping (for classification="expense")
Map to:

* bucket="marketing" if contains: marketing, ads, acquisition, growth, brand, performance, ua
* bucket="payroll" if contains: salary, payroll, wages, people, contractor, benefits
* bucket="cogs" if contains: cogs, payout, commission, astrologer, payment processing, fulfillment, logistics, packaging
* else bucket="operating"

Special case mappings for Astrotalk style:

* "Astrologer Expenses" → cogs
* "Logistics And Packaging" → cogs unless user toggles to operating
* "Astromall Expenses" → cogs by default

B5) Normalization Rules (NON-NEGOTIABLE)
Canonical storage:

* For classification="expense": normalized_value = abs(raw_value)
* For classification="revenue": normalized_value = abs(raw_value) (but warn if raw is negative)
* For derived/header/other: keep raw_value as-is but DO NOT use for totals unless user includes explicitly

B6) Month Selection
If time series exists:

* Default selected_period_mode="latest" and selected_period=latest completed month in file
* Offer "avg_3mo": compute 3-month rolling average for baseline metrics

C) BASELINE METRICS + BURN/RUNWAY FORMULAS (Fix)
================================================

Implement baseline computation for selected period (or avg_3mo):

* revenue = sum(all included revenue rows for that period) OR user-selected primary revenue row if multiple candidates exist
* total_expenses = sum(all included expense rows (normalized) for that period)
* net_burn = total_expenses - revenue
* status:

  * if net_burn < 0 → "Profitable / Surplus"
  * else → "Burning Cash"
* runway:

  * if net_burn <= 0 → runway="Sustainable"
  * else runway_months = cash_on_hand / net_burn (require cash_on_hand; if missing show warning and set runway_months=null)

IMPORTANT UI RULE:
Never display negative expenses. Ever.
Display burn as:

* “Monthly Burn” if net_burn > 0 (value positive)
* “Monthly Surplus” if net_burn < 0 (display abs(net_burn))

Astrotalk post-fix expected behavior:
For the latest month in the given Excel format, total expenses should be positive and net_burn likely negative (surplus). UI should show “Sustainable” not “36 months runway” if profitable.
