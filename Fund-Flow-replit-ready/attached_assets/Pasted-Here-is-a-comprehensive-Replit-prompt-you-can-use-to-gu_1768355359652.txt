Here is a comprehensive Replit prompt you can use to guide an AI coding assistant (or a collaborating developer) to implement the “easy‑to‑understand” UX enhancements for your Scenario Builder. This prompt assumes a React/Next.js environment and aims to be explicit about desired behaviors, components, and user flows.

---

**Replit Prompt – Implement Simplified, Contextual UX for Scenario Builder**

I am enhancing the UX of our startup financial simulation tool (Scenario Builder) hosted at `/scenarios`. The current wizard has sliders for Pricing Change, Growth Uplift, and Burn Reduction, but it feels technical and is hard for non‑financial users to interpret. I want to redesign this screen to provide clearer context and guidance. Please help me refactor/extend the existing codebase with the following detailed requirements:

---

### 1. Plain‑Language Descriptions and Narrative Feedback

* **Add explanatory text beneath or adjacent to each slider** that uses simple, non‑technical language. For instance:

  * *Pricing Change*: “Adjust how much you charge for your product or service. A +5 % change means increasing the price from $100 to $105.”
  * *Growth Uplift*: “Estimate how fast your customer base will grow compared to today. A +5 % growth uplift means slightly faster customer acquisition.”
  * *Burn Reduction*: “Lower your monthly spending by streamlining operations. A –5 % reduction means spending $95 instead of $100.”
* **Implement a narrative feedback generator**: As the user moves any slider, display a one‑sentence summary (e.g., in a banner or a side panel) like “You’ve increased your price moderately and expect your business to grow faster; your runway is projected to extend by 2.3 months.” This should update live with each change.

### 2. Visual Aids and Consistent Iconography

* **Introduce icons** next to each slider label:

  * Pricing Change – price tag icon.
  * Growth Uplift – upward‑trending arrow or rocket icon.
  * Burn Reduction – scissors or flame icon with a downward arrow.
* **Colour coding**:

  * Use a consistent primary colour (e.g., green) to indicate “increase” sliders (Pricing Change, Growth Uplift).
  * Use another (e.g., red or orange) to indicate “decrease” sliders (Burn Reduction).
  * Make sure colours are accessible (sufficient contrast) and have CSS variables that can be adjusted in the theme.
* **Add a progress indicator** above the wizard steps. This can be a bar or numbered checklist showing which step the user is on and which steps remain.

### 3. Real‑Time Metrics Dashboard and Summary Card

* **Create a `MiniDashboard` component** that displays at least these metrics:

  * Current Runway (from baseline company data).
  * Projected Runway (after applying slider changes).
  * Estimated monthly revenue and burn after adjustments.
* The dashboard should update in real-time when slider values change. It should be visually separated (e.g., card with shadow) and placed either on the right side or below the sliders on smaller screens.
* **Implement a `SummaryCard` component** that appears after a user has completed a step. For example, when leaving the Parameters step, show a card summarizing: “You selected a +5 % price increase, +5 % growth uplift, and –5 % burn reduction. Your runway increases from 15.3 to 17.6 months.” Provide a clear CTA like “Continue to Fundraising” or “Save this Scenario.”

### 4. Guided Workflow and Tooltips

* **Break the “Adjust Parameters” page into sub-sections** or use a stepper to show only one slider at a time. After a user adjusts the first slider (e.g., Pricing Change), present a “Next” button or automatically transition to the next slider with a brief description.
* **Tooltips**: For each slider, add a small `(i)` icon or similar. On hover or click, display a tooltip containing:

  * A definition (“Burn Reduction represents the percentage you expect to lower your monthly operating expenses.”).
  * Reasonable value ranges (e.g., “Typical pricing changes range from –10 % to +20 %”).
  * An example in plain terms (see requirement 1).
* Ensure tooltips are accessible: they should be reachable via keyboard (focus/enter) and readable by screen readers (use `aria-label` and `aria-describedby`).

### 5. Scenario Templates and Presets

* **Implement three preset buttons** at the top of the Parameters step: “Conservative”, “Balanced”, and “Aggressive”. Each preset applies default values to the sliders:

  * Conservative: –5 % pricing change, 0 % growth uplift, –10 % burn reduction.
  * Balanced: +5 % pricing change, +5 % growth uplift, –5 % burn reduction.
  * Aggressive: +10 % pricing change, +15 % growth uplift, –2 % burn reduction.
* After a preset is selected, show a text explanation of what it means (e.g., “Balanced means moderate price increases with steady growth expectations.”), and still allow the user to adjust the sliders further.

### 6. Accessibility and Responsiveness

* Ensure all sliders are fully keyboard accessible. Users should be able to tab into a slider and use arrow keys or type values in an adjacent text field.
* For each slider, provide an **optional numeric input box** so users can directly enter the desired percentage.
* Ensure the entire layout is responsive: on small screens, place the mini dashboard below the sliders; on larger screens, use a two‑column layout with sliders on the left and the dashboard/summary on the right.

### 7. Implement a ‘Context Sidebar’ (optional but recommended)

* Alongside the mini dashboard, create a sidebar that lists:

  * Selected template (if any).
  * Current slider values with colour-coded icons.
  * The narrative summary sentence.
* This sidebar should persist as the user navigates through the wizard steps, giving them an at‑a‑glance overview of their assumptions.

### 8. Testing and Validation

* Add basic input validation for slider ranges (e.g., prevent values beyond a sensible domain such as –50 % to +200 %).
* Write unit tests (if using Jest/React Testing Library) to verify:

  * The narrative feedback updates correctly when sliders change.
  * Preset buttons assign the correct default values.
  * Tooltips contain expected content and are accessible via keyboard.
  * The mini dashboard recalculates metrics based on slider inputs.

---

Please integrate these changes into the existing codebase, creating new components as necessary (e.g., `MiniDashboard`, `SummaryCard`, `PresetButtons`). Make sure all new UI elements align with our design system or theme (use existing CSS modules/styled-components where appropriate). The goal is to deliver a clear, contextual, and supportive experience that helps non‑technical founders understand how pricing, growth, and spending assumptions affect their runway and overall financial health.
