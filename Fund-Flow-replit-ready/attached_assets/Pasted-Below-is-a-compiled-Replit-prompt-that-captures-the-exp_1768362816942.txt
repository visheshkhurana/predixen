# Project: Enhanced Admin Dashboard for Fund Flow / Predixen

## Overview
We need to build a robust admin dashboard that empowers super‑admins to manage users, companies, subscriptions, templates, alerts and more. The dashboard should offer granular role/permission controls, rich analytics and a polished user experience. Additionally, we want new features like bulk actions, impersonation, global notifications, report exports and a customizable UI.

## Feature Requirements

1. **User & Role Management**
   - List all users with avatars, status badges, roles and last login.
   - Invite new users; bulk suspend/reactivate or change roles.
   - Role & permission matrix to allow fine‑grained access control.
   - Impersonation mode for admins to view the app as a specific user (with audit logging).

2. **Company Directory**
   - Show all companies/tenants with summary metrics (runway, growth index, burn multiple).
   - Edit company information (name, stage, industry, description, employees).
   - Link to each company’s simulations, decisions and alerts.

3. **Subscription & Billing**
   - Integrate Stripe (or Paddle) to display current plan, renewal dates, seat counts.
   - Upgrade/downgrade plans; handle payment failures; issue credits/refunds.
   - Export invoice history for accounting.

4. **Template & Scenario Management**
   - Central library of simulation templates: create, duplicate, edit, retire and categorize.
   - Show template usage statistics.

5. **Notifications & Alerts**
   - Global notifications center combining anomalies, threshold breaches, failed simulations and payment issues.
   - Filter alerts by severity, company, date.
   - Configurable alert thresholds and defaults per plan.

6. **Login History & Audit Trail**
   - Log every login with IP, device, browser, timestamp and success/failure.
   - Searchable audit trail of admin actions with filters by user, action type and date.

7. **Reports & Exports**
   - Generate CSV/Excel reports for user lists, login history, subscription details, or aggregated metrics.
   - Schedule periodic exports via email.

8. **Support & Ticket Integration**
   - Pull in tickets from a helpdesk system (Zendesk, Intercom) and associate them with users/companies.

9. **Customization & Preferences**
   - Customizable dashboard layout (add/remove KPI cards, reorder).
   - Theme toggle (dark/light) with saved preference.
   - Configure default templates and alert thresholds by subscription tier.

## Architecture Recommendations

### Tech Stack
- **Frontend:** Next.js (React) with TypeScript, Tailwind CSS, Radix UI or Headless UI for accessible components. Zustand for state management.
- **API Layer:** tRPC or RESTful routes via Next.js API or a separate FastAPI/Express service, using JWT‑based auth with NextAuth.js.
- **Database:** PostgreSQL with Prisma ORM. Tables: `User`, `Company`, `Subscription`, `Template`, `LoginHistory`, `AuditLog`, `Notification`, etc.
- **Payments:** Stripe with product/price IDs configured per plan. Webhook endpoint at `/api/webhooks/stripe`.
- **Auth & RBAC:** Use NextAuth.js sessions; extend session object with `role` and `permissions` fields; implement middleware for route protection.
- **Hosting:** Continue using Replit’s deployment platform; store secrets via Replit secrets.

### Sample Code Snippet: Notification API Route

```ts
// pages/api/admin/notifications/index.ts
import { NextApiRequest, NextApiResponse } from 'next';
import { getServerSession } from 'next-auth';
import prisma from '../../../../lib/prisma';
import { isAdmin } from '../../../../lib/auth';

export default async function handler(req: NextApiRequest, res: NextApiResponse) {
  const session = await getServerSession(req, res);
  if (!session || !isAdmin(session.user)) {
    return res.status(403).json({ error: 'Forbidden' });
  }
  if (req.method === 'GET') {
    const { severity, unread } = req.query;
    const where: any = {};
    if (severity) where.severity = severity;
    if (unread) where.read = unread === 'true';
    const notifications = await prisma.notification.findMany({
      where,
      orderBy: { createdAt: 'desc' },
      take: 100,
    });
    return res.json(notifications);
  }
  res.setHeader('Allow', ['GET']);
  return res.status(405).end(`Method ${req.method} Not Allowed`);
}
```

1. **Dashboard Home:**
