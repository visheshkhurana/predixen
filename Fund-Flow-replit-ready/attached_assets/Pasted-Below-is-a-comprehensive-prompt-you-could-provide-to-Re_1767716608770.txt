Below is a comprehensive prompt you could provide to Replit’s AI (or a developer working on the project) to enhance the **Simulation** module of Predixen Intelligence OS. It builds on the observations and suggestions from testing and is structured to guide the developer toward actionable improvements.

---

### 🧩 Project Context

You are enhancing the **Simulation** module of Predixen Intelligence OS, an AI‑powered financial planning tool for startups. The current simulation page allows users to adjust three sliders (pricing change, growth uplift, burn reduction) and optional fundraising inputs, run projections, and save scenarios. There are four default templates (“Conservative Cut,” “Growth Push,” “Price Increase,” “Bridge Round”). Users can run simulations and see revenue and cash‑balance projection charts, but duplication issues and limited presets hamper usability.

### 🚀 Goal of This Task

Improve the Simulation module’s utility and user experience by:

1. Expanding the number and variety of pre‑configured scenario templates (aim for 5–10).
2. Enhancing scenario management (e.g., prevent duplicates, enable comparison).
3. Improving the user interface with interactive charts and clear summaries.
4. Providing contextual guidance on how parameter changes affect metrics.

### 🔨 What Needs To Be Built / Modified

1. **Add More Built‑in Templates (5–10 total)**

   * Introduce new template definitions in the codebase. Suggested scenarios:

     * **Baseline (Status Quo)** – no adjustments; serves as a control.
     * **Moderate Growth Push** – +5 % growth uplift, –5 % burn reduction.
     * **Aggressive Growth** – +15 % growth uplift, –5 % burn reduction.
     * **Cost‑Cut Scenario** – –25 % burn reduction (shows impact on runway).
     * **Price Increase** – +10 % pricing change.
     * **Fundraising Delay** – shift fundraising month out by 3 months and keep other inputs neutral.
     * **Best Case** – high growth uplift (+20 %), moderate price increase (+5 %) and burn reduction (+10 %).
     * **Worst Case** – negative growth (–10 %), no burn reduction, fundraising delayed.
   * Each template should populate the slider values and scenario name accordingly.

2. **Scenario Management Enhancements**

   * **Prevent Duplicates:** When a user runs a template with the default name and the name already exists in `Saved Scenarios`, prompt the user to rename it or overwrite the existing entry.
   * **Tagging & Filtering:** Add metadata (e.g., tags like `growth`, `cost-cutting`, `fundraising`) to scenarios; implement simple UI filters to view only certain tags.
   * **Comparison Table:** Create a component that lists saved scenarios side‑by‑side. For each scenario, display key outputs such as:

     * P50 runway (months)
     * End-of-horizon cash balance
     * Average monthly net burn
     * Revenue growth rate
       This table will let users quickly compare the effect of different strategies.

3. **UI & UX Improvements**

   * **Interactive Charts:** Replace static images with interactive charts (e.g., using Chart.js or D3). Enable hover tooltips that show exact P10/P50/P90 values at each month.
   * **Summary Banner:** After a simulation runs, display a succinct summary at the top of the results: e.g., “Runway P50 extends to 21 months; probability of surviving 18 months is 98 %.”
   * **Multi‑Factor Adjustments:** Allow users to set multiple parameters simultaneously in a single scenario (e.g., adjust pricing, growth uplift and burn reduction at once) rather than limiting to one template per adjustment.

4. **Contextual Guidance**

   * Add small info buttons or tooltips next to each slider that describe typical ranges and their expected impact. For instance, hovering over the “Growth Uplift” slider could show: “+5 % is conservative; +10 % moderate; +20 % aggressive,” with a note on how growth uplift interacts with revenue projections.
   * Provide an explanation modal that opens when the user clicks a “Learn more about simulations” link. This modal should cover how P10/P50/P90 are derived, assumptions behind projections and tips on interpreting results.

### 🧠 Implementation Notes

* **State Management:** Use your existing state management solution (e.g., React Context/Zustand) to store templates, saved scenarios and tags. Ensure scenario updates propagate across the comparison table and charts.
* **Data Persistence:** Save scenarios to backend storage or local storage so users don’t lose them on page refresh. Tag metadata should persist as well.
* **Component Structure:** Consider breaking the simulation page into components: `TemplateSelector`, `ScenarioForm`, `ProjectionCharts`, `ScenarioSummary`, `ScenarioComparison`, etc. This will make it easier to maintain and extend.
* **Accessibility:** Add ARIA labels to sliders and interactive elements. Ensure keyboard navigation works for adjusting values and selecting templates.

### ✅ Success Criteria

* The Simulation page lists 5–10 varied built‑in templates, each pre‑configured with different parameter settings.
* Running a template creates (or overwrites) a named scenario without cluttering the Saved Scenarios list with duplicates.
* Users can filter or search the Saved Scenarios list and view a comparison table summarizing key metrics across scenarios.
* Charts are interactive, with hover tooltips and clear legends, and each simulation result shows a concise summary banner.
* Users understand how adjustments affect outcomes thanks to tooltips and explanatory modals.

Providing this prompt to Replit’s AI or your developer should give them clear direction on the desired features and the specific changes needed to make the simulation module richer, easier to use and more aligned with having 5–10 ready‑made scenarios.
