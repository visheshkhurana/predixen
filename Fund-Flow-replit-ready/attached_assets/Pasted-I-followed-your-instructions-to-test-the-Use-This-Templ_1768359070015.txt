### Observations

* The application successfully created the scenario (as evidenced by the toast), but it redirected to a page that is not registered in the router.
* The error message suggests that there is no corresponding route handler (likely in a framework like Next.js) for the new scenario URL.

### Suggested Fix

To resolve this issue, ensure that your routing configuration includes a page for individual scenarios. In a Next.js project, this typically means adding a dynamic route under `pages/scenarios/[id].jsx` or `pages/scenarios/[id]/index.jsx`. The page should fetch the scenario data by ID and render the appropriate details.

**Steps to fix:**

1. **Define a dynamic route**: Add a file such as `pages/scenarios/[id].tsx` that uses `useRouter()` to read the `id` parameter from the URL.
2. **Fetch scenario data**: In this page component, fetch the scenario data from your API or context store using the `id` and display it.
3. **Update routing configuration**: If you have a custom router or are using a framework other than Next.js, ensure that the `/scenarios/:id` route is registered and points to the correct component.
4. **Handle missing scenarios gracefully**: If a scenario ID does not exist or is invalid, display an informative error page (e.g., “Scenario not found”).

Implementing a proper dynamic route will allow users to view newly created scenarios without encountering this error message.
