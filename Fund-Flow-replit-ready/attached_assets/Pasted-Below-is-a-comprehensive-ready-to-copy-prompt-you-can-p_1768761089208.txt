**Prompt: Improve the Decision Recommendations Page UI/UX**

> **Context**
> We have a Decision Recommendations page that displays three strategic options (e.g., “#1 Recommended: Product‑Led Growth Investment,” “#2 Alternative: Fundraise in 3 Months,” etc.). Each card currently lists a title, survival impact, runway change, a brief description, and links to hidden details like rationale, key assumptions and risks.
>
> **Goal**
> Redesign this page so that CXOs and founders can quickly understand, compare, and act on these decisions. The UI should be intuitive, visually engaging, and provide enough context (metrics, visual cues, and interactive elements) to support data‑driven decision‑making.

### Tasks

1. **Enhance Visual Hierarchy**

   * Increase the prominence of each card’s header (e.g., “#1 Recommended”). Use larger fonts and bold text.
   * Apply distinct colors or icons to each card to differentiate the recommended option from alternatives.

2. **Improve Expand/Collapse Interactions**

   * Replace the small, subtle collapse arrow with a clearly labelled “View details” button or an obvious accordion header.
   * When expanded, show the rationale, key assumptions and risks in a well‑spaced layout using bullet points or numbered lists.

3. **Add Visual Comparison Aids**

   * For each card, create small horizontal bar charts or progress indicators to visually represent:

     * **Survival Impact** (e.g., 18 months) — display a bar proportional to the longest impact across all options.
     * **Runway Change** — use color coding (green for runway extension, red for reduction) so differences are clear at a glance.
   * Ensure that these bars update based on the actual numeric values supplied by the backend.

4. **Introduce Interactive Controls**

   * Add sliders or editable numeric inputs on each card allowing users to tweak key assumptions (e.g., amount raised, growth rate, cost reduction).
   * When a user modifies an assumption, recalculate and update the displayed metrics (survival impact, runway change) in real time.

5. **Provide Tooltips and Help**

   * Implement hover (or tap) tooltips for terms like “Survival Impact,” “Runway Change” and “Risks & Assumptions” explaining what they mean and why they are important.
   * Include a “Learn More” link or icon that opens a modal with in‑depth documentation or examples.

6. **Add Call‑to‑Action Buttons**

   * On the recommended option, include a primary CTA such as “Adopt Plan” or “Run Scenario” that triggers the next step in your app (e.g., launching a simulation or generating a report).
   * On alternative options, provide secondary actions like “Refine Assumptions” or “Compare Further.”

7. **Augment Decision Data (Optional but Recommended)**

   * In addition to survival/runway metrics, display other useful KPIs if available: projected revenue growth, EBITDA impact, or cumulative cash burn.
   * Include a simple timeline or milestone view showing major events (e.g., fundraising rounds, product launches) associated with each decision.
   * If possible, show how each option aligns with strategic objectives or how it compares against industry benchmarks.

### Deliverables

* Updated UI components implementing the above features.
* Styling and layout updates (CSS/SCSS/tailwind) to support the new visual hierarchy and comparison aids.
* Front‑end logic (React/Vue/Svelte) to handle interactive assumption sliders and dynamic recalculation of metrics.
* Accessibility support (ARIA labels, keyboard navigation for collapsible sections and sliders).
* Unit tests covering new UI components and interactive functionality.

### Constraints

* Maintain responsiveness for both desktop and mobile layouts.
* Do not alter existing backend APIs or data structures; enhancements should consume the existing decision data.
* Ensure all charts use a neutral color palette unless specifically indicating positive/negative outcomes (green/red).
* Preserve a clean, professional aesthetic consistent with the rest of the application.
