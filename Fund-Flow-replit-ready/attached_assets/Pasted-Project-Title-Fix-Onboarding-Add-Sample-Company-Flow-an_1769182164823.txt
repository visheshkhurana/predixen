Project Title: Fix Onboarding “Add Sample Company” Flow and Truth Scan Trigger

Overview:
Users who select the sample company flow get stuck on Step 2 of onboarding.  After filling company details and pressing Continue, the wizard advances to the Financial Baseline step, but clicking “Run First Truth Scan” does nothing.  This bug prevents users from completing onboarding.  We need to ensure the button triggers the scan API and advances to Step 3 (Growth Targets / Results) while providing feedback.

Tasks:

1. **Audit Stepper State Management**
   - Inspect the component handling the onboarding wizard (likely in `/pages/onboarding.jsx` or similar).  Identify the state variable controlling the step index (e.g., `currentStep`).
   - Ensure that clicking the Step 2 submit button dispatches an action or updates state to proceed to Step 3 after the scan is initiated.

2. **Hook Up the Scan API**
   - Implement or verify the API call that runs the Truth Scan for sample data.  This should be an async function (e.g., `runSampleTruthScan()`) that:
     - Sends the sample financial values (revenue, gross margin, opex, payroll, other costs, cash balance) to the backend endpoint (`/api/truth-scan` or equivalent).
     - Waits for a response (success or error).
   - If the response is successful, update the wizard’s step state to advance to Step 3 and persist the returned scan results in context/store.
   - If there’s an error (e.g., network failure), display a toast or inline error explaining that the scan could not be run, with an option to retry.

3. **Provide UI Feedback**
   - Show a loading spinner or disable the “Run First Truth Scan” button while the scan API call is in progress.
   - On success, clear the loading state and navigate to Step 3.
   - On failure, re‑enable the button and display an error message.

4. **Validate Sample Inputs**
   - Even though sample values are pre‑filled, ensure that the form’s `required` attributes are respected.  If a user clears a field, prevent submission and display a validation error.
   - Consider grey‑out or hide the number inputs for sample mode to indicate they are immutable defaults.

5. **Update Step 3 (Growth Targets)**
   - Verify that Step 3 renders correctly after the Truth Scan runs.  It should display the baseline metrics and allow users to input growth targets.
   - If Step 3 relies on scan results that aren’t populated for sample companies, populate them with defaults or handle missing data gracefully.

6. **Testing**
   - Add unit tests for the onboarding component to ensure that:
     - Clicking Continue after filling Step 1 advances to Step 2.
     - Clicking Run Scan triggers the API call and advances to Step 3 on success.
     - Appropriate error messages appear on failure.
   - Perform manual testing in the browser to ensure the sample company flow completes and populates subsequent pages.

7. **Optional Enhancement: Sample Company Shortcut**
   - On the Step 1 page, provide a “Load Sample Company” button that bypasses manual entry and fills in sample company name, industry and stage.
   - When clicked, automatically set the form values and proceed to Step 2, where sample financials are prefilled.

Deliverables:
- Updated onboarding component with working step transitions.
- Backend or mock API call for running sample Truth Scans.
- User feedback (loading indicators, error handling) during scan.
- Tests verifying the flow works correctly.

Success Criteria:
- Users who select or enter a sample company can click Continue and then Run First Truth Scan and proceed to Step 3 without errors.
- The wizard provides clear feedback on progress and handles errors gracefully.
- Manual and unit tests confirm the fix.
