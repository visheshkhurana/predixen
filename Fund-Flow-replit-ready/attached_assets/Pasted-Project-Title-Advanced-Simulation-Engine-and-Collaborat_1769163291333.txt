Project Title: Advanced Simulation Engine and Collaborative Scenario Planning

Overview:
We have redesigned the core simulation backend to support probabilistic modelling, modular data ingestion and an API‑first architecture.  This project extends that work to add deeper analytics (sensitivity analysis, goal‑seeking), scenario management features (versioning, templates, narratives), macro‑economic modifiers, optional machine‑learning forecasts, and collaboration tools.  The goal is to create a decision‑support platform that founders and their teams can rely on for strategic planning.

### 1. Sensitivity Analysis

- After each simulation run (Monte Carlo), compute the partial derivatives of key outputs (runway, survival probability, ARR) with respect to each input variable.
- Implement a function to rank variables by their influence on the outcome.  For example, vary each assumption ±10 % while keeping others constant to observe changes in outputs.
- Provide an API endpoint `GET /sensitivity/{run_id}` that returns the sensitivity metrics.
- In the UI, visualise this using a tornado or bar chart so users can see which assumptions matter most.

### 2. Scenario Versioning & History

- Extend the scenario model to include a `version` and `created_at` timestamp.  When a user saves a scenario, store the current assumption set and simulation results.
- Provide endpoints to:
  - `GET /scenarios` → list all scenarios with version history.
  - `GET /scenarios/{scenario_id}/versions` → list versions of a specific scenario.
  - `POST /scenarios/{scenario_id}/clone` → create a new version with updated assumptions.
- Allow the UI to compare versions of a scenario and show how outcomes change over time.

### 3. Macro‑Economic Modifiers

- Define a global “macro environment” object with parameters like `interest_rate`, `inflation_rate`, `market_growth_factor`.
- Allow users to choose from presets (e.g., “Optimistic”, “Neutral”, “Pessimistic”) or customise these values.
- Modify revenue and cost projections based on these factors (e.g., high inflation increases costs faster).
- Expose an endpoint `GET /macro` and `PUT /macro` for retrieving/updating presets.

### 4. ML‑Driven Forecasts (Optional)

- Integrate a simple forecasting module that fits time‑series models (Prophet, ARIMA) on historical revenue and cost data.
- Use the forecasts as default distributions for revenue growth and expense inflation.
- Provide an option in the API to switch between “manual assumptions” and “ML forecasts” for baseline projections.

### 5. Action Recommendations

- After running a simulation, evaluate whether the outputs meet typical health checks (e.g., runway ≥ 12 months, burn multiple ≤ 2).
- If not, generate suggestions such as “Reduce burn by X %”, “Delay hiring by Y months”, or “Increase pricing by Z %”.
- These recommendations can leverage the optimisation module: run small searches varying key variables until targets are met.
- Include the recommendations in the simulation results payload and expose via `GET /recommendations/{run_id}`.

### 6. Constraint Handling and Multi‑Metric Objectives
