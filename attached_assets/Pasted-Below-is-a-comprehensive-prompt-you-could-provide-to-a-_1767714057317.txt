### 🔧 Project Overview

You are working on **Predixen Intelligence OS**, an AI‑powered financial intelligence tool for startup founders. The current MVP includes onboarding, a Truth Scan dashboard, simulation tools, a decision recommendation engine, and an AI Copilot chat. Users can create a company, enter baseline financial data, run preset or custom simulations, and ask the Copilot for advice.

### 🎯 Goals & Key Improvements

The aim is to enhance usability, transparency, and decision‑making power. Implement the following feature upgrades:

#### 1. **Contextual Tooltips & Metric Explanations**

* Add tooltip icons (`🛈`) next to each baseline input (revenue, gross margin, opex, payroll, etc.) explaining why the metric matters and how it feeds into downstream calculations.
* In the Truth Scan dashboard, allow users to click on metrics like **Burn Multiple** or **Operating Margin** to open a modal or sidebar that explains the formula and shows a breakdown (e.g., a table of revenue vs. expenses over time).
* Provide industry comparisons in these modals, highlighting how the user’s metrics compare to quartiles.

#### 2. **Enhanced Simulation Capabilities**

* Refactor the simulation engine to accept multiple simultaneous adjustments (e.g., burn reduction **and** price increase **and** fundraising). Users should be able to define all three sliders in one scenario.
* Offer guidance for each slider by adding annotated markers on the slider track—for example, “Conservative (–10 %), Moderate (0 %), Aggressive (+10 %)”.
* Allow users to specify their own probabilistic ranges instead of fixed P10/P50/P90 (e.g., enter custom pessimistic, base, and optimistic values or use a dropdown to choose different percentiles).
* After a simulation runs, display results in interactive charts (using a library like Chart.js or D3) that allow hovering to see exact values at each month.

#### 3. **Decision Recommendation Enhancements**

* Ensure decisions are automatically generated after every simulation. If none are generated, display an explanation (e.g., “Insufficient data; please run more varied scenarios”).
* Create a `Decisions` page that lists recommendations with their expected impact (e.g., extend runway by X months) and rationale.
* Implement a “History” section so users can see past recommendations, mark them as accepted/rejected, and track implementation status.
* Add filters or tags (e.g., Cost‑cutting, Growth, Fundraising) so users can find relevant decisions quickly.

#### 4. **Copilot Improvements**

* In Copilot chat, allow users to edit suggested parameter values before launching a new simulation. For example, after Copilot recommends a 10 % price increase, show editable chips the user can change (e.g., 8 %) prior to clicking “Run simulation”.
* Provide a quick way to run the suggested scenario by embedding a button in Copilot’s answer, which pre-fills the simulation form with those parameters.
* Add a thumbs‑up/thumbs‑down or short text feedback option after each Copilot response to collect user sentiment. Store this feedback to inform future AI responses.

#### 5. **User Experience & Miscellaneous**

* Display progress/loading indicators when simulations or decision generation are running.
* Permit managing multiple companies under one account (e.g., via a dropdown in the sidebar).
* Add an export feature that allows users to download their simulations, Truth Scan metrics, or decision history as CSV or PDF.
* Ensure all UI elements are responsive and accessible (consider keyboard navigation and ARIA labels).

### 🛠 Tech Considerations

* The app appears to be built with a modern web stack (likely React/Next.js). Maintain consistent code style and component architecture.
* Use state management (e.g., Zustand, Redux, React Context) to store metrics, simulation parameters and decision history.
* Make backend/API changes to support multi‑factor simulations and persisting decision feedback. Ensure data schemas account for multiple companies per user.

### ✅ Acceptance Criteria

* Users see informative tooltips throughout onboarding and dashboards.
* Clicking on any metric in Truth Scan opens a detail view explaining the metric and showing breakdowns.
* Simulations accept multiple adjustments and produce interactive charts with hoverable tooltips.
* Decisions are generated automatically and list actionable recommendations with rationales; users can view past decisions and mark them as accepted/rejected.
* Copilot responses allow parameter editing and quick scenario execution, and users can give feedback on the advice.
* The application supports multiple companies, displays loading indicators during long operations, and provides data export.

By following this prompt, the Replit AI or developer will understand the current context, the desired enhancements, and the expected outcome for each feature.
