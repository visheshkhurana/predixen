Predixen UX/UI Improvement Prompt for Replit
Project Overview
Improve the UX/UI of Predixen, a startup financial intelligence platform. The goal is to make the application super intuitive and easy to use for all types of users (technical founders, non-technical founders, finance teams, investors).

GLOBAL IMPROVEMENTS
1. Navigation & Information Architecture
Current Issues:

Left sidebar has 14+ menu items creating cognitive overload
No clear grouping or hierarchy between features
Company selector dropdown is positioned oddly at top of nav
Top stepper (Truth → Simulation → Decision) competes with left nav for primary navigation
Improvements:

Group navigation items into logical sections with collapsible headers:
Data & Setup: Data Input, Integrations, Marketplace
Analytics: Overview, KPI Board, Truth Scan, Alerts
Planning: Simulation, Templates, Decisions, Scenarios
Stakeholders: Fundraising, Investor Room, KPI Dashboards
Add visual separators between groups
Make the top workflow stepper more prominent OR remove it and integrate progress into sidebar
Move company selector to a persistent top bar with breadcrumb navigation
Add keyboard shortcuts overlay (press "?" to show)
Implement command palette (Cmd+K) for power users to quickly navigate
2. Onboarding & Empty States
Current Issues:

Empty states (Cap Tables, Dashboards, Alerts) show minimal guidance
No clear "getting started" flow for new users
Tutorial popup on Classic Wizard is easily dismissed and not contextual
Improvements:

Design engaging empty states with:
Illustration + clear value proposition
Primary CTA button
"Learn more" link to documentation
Sample/demo data option ("See example")
Create progressive onboarding checklist that persists across sessions
Add contextual tooltips on first visit to each section
Implement "Quick Setup Wizard" that guides through: Company Info → Financial Data → Goals → First Simulation
3. Visual Hierarchy & Typography
Current Issues:

Page titles all look the same weight/style
Subtitles and descriptions blend together
Card-based layouts lack clear visual rhythm
Status indicators (Critical, Healthy, Caution) need more consistency
Improvements:

Establish clear typography scale:
Page titles: 28px bold
Section headers: 20px semibold
Card titles: 16px medium
Body text: 14px regular
Labels: 12px medium uppercase with letter-spacing
Add more whitespace between sections (increase from 16px to 24px or 32px)
Use subtle background color shifts to delineate sections
Standardize status badge system across all pages with consistent colors and shapes
4. Dark Theme Refinements
Current Issues:

Some contrast issues (gray text on dark backgrounds)
Cards blend into background
Blue accent color sometimes hard to see
Improvements:

Increase card background lightness slightly (from current 
#1C1C1E to 
#242428)
Add subtle 1px border or shadow to cards for better definition
Ensure all text meets WCAG AA contrast requirements
Use brighter accent colors for interactive elements
Add subtle gradient or pattern to hero sections for visual interest

"Save All Data" button at bottom may be missed
Improvements:

Reorganize tabs:
Manual Entry: Company, Financials, Goals
Import: CSV, PDF, Excel (as secondary option or separate section)
Add circular progress indicator showing "Profile 60% complete"
Make AI Assistant collapsible by default, expandable on click
Add inline validation with real-time feedback
Sticky save button that appears when user scrolls
Add autosave functionality with "Last saved 2 minutes ago" indicator
Show field-level help tooltips on hover/focus
Add smart defaults and placeholders with realistic example data
