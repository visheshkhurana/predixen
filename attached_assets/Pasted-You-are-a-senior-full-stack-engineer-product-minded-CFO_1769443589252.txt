# 1) Implement canonical Financial Model (single source of truth)

## 1.1 Create a canonical schema

Create `src/lib/finance/schema.ts`:

* `NormalizedMonthlyFinancials`:

  * `period: { year: number; month: number; label: string }`
  * `revenue: number`
  * `expenses: { cogs: number; marketing: number; payroll: number | null; operating: number; other: number }`
  * `totalExpenses: number`
  * `netBurn: number`  // **expenses - revenue** (positive = burning, negative = profitable)
  * `runwayMonths: number | null` // if netBurn > 0 and cashOnHand > 0 => cash/netBurn else null
  * `isProfitable: boolean` // revenue >= expenses
  * `sources: { [key: string]: { sheet?: string; row?: string; cellRange?: string } }` // traceability
  * `validation: { warnings: ValidationWarning[]; errors: ValidationError[] }`

Define:

* `type ValidationSeverity = "info" | "warn" | "error"`
* `ValidationWarning = { code: string; message: string; fields: string[]; severity: ValidationSeverity }`
* `ValidationError = { code: string; message: string; fields: string[]; severity: "error" }`

## 1.2 Define sign conventions (must be consistent everywhere)

* Store all costs as **positive numbers** in the model.
* `netBurn = totalExpenses - revenue`

  * `netBurn > 0` => burning cash
  * `netBurn < 0` => net positive (profitable)
* UI must label properly:

  * If `netBurn < 0` show “Net Positive: +$X/mo” where X = abs(netBurn)
  * If `netBurn > 0` show “Net Burn: -$X/mo” or “Burn: $X/mo” (choose one and be consistent)
* Runway:

  * If `netBurn <= 0` => show “Profitable / Sustainable” and runwayMonths = null
  * Else runwayMonths = cashOnHand / netBurn

# 2) Fix Excel ingestion + mapping (root cause)

We are missing marketing aggregation and fabricating payroll. Implement robust mapping.

## 2.1 Ingestion entrypoint

Create `src/lib/finance/importers/excelImporter.ts` that:

* uses `xlsx` to read workbook
* detects candidate sheets (“Income Statement”, “P&L”, “Profit and Loss”, etc.)
* extracts monthly columns (dates) and row labels
* returns a raw table: `rows: { label: string; valuesByPeriod: Record<string, number> }[]`

## 2.2 Normalization + mapping

Create `src/lib/finance/normalize/mapPnLToCanonical.ts`:

* Input: raw rows + target period (default latest)
* Output: `NormalizedMonthlyFinancials`

### Mapping rules (IMPORTANT)

* Revenue:

  * match rows containing: “Total Revenue”, “Revenue”, “Net Revenue”, “Sales”
* COGS:

  * match: “COGS”, “Cost of Goods”, “Cost of revenue”, “Direct costs”
* Marketing:

  * sum ALL rows matching:

    * “Marketing”
    * “Sales & Marketing”
    * “User acquisition”
    * “Brand”
    * “Performance marketing”
    * “India Marketing”, “International Marketing”, etc.
  * **Do not** pick only one sub-row; aggregate all marketing-related rows.
* Payroll:

  * match only if row label contains:

    * “Payroll”, “Salaries”, “Wages”, “Employee costs”, “Compensation”
  * If no payroll row present => `payroll = null` and add validation warning `PAYROLL_NOT_FOUND`.
  * Never default payroll to 50K or any constant.
* Operating:

  * match “Operating”, “G&A”, “General & Admin”, “Administration”, “Office”
* Other:

  * match “Other”, “Misc”, “Utilities”, etc.
* Total expenses:

  * compute = cogs + marketing + (payroll ?? 0) + operating + other
  * If raw sheet has “Total Expenses” row, compare and warn if mismatch > 2%

### Traceability

For each mapped field, populate `sources` pointing to which rows/cells contributed (sheet + row label + period key).

# 5) Validation + anomaly detection (prevent similar errors)

Create `src/lib/finance/validate/validators.ts` and run validators after normalization:

### Required fields checks

* revenue must be >= 0
* cogs/marketing/operating/other >= 0
* totalExpenses = sum components (within small epsilon)

### Anomaly rules (warn)

* If `marketing > revenue * 0.8` => WARN (possible mapping issue)
* If `marketing < revenue * 0.001` and revenue > 1M => WARN (likely missing marketing)
* If payroll is null => WARN “Payroll not detected”
* If netBurn magnitude is suspicious:

  * abs(netBurn) > revenue * 2 => WARN
* If any category is 0 but raw sheet had matching rows => WARN

### UI display for validation

* show warnings/errors with codes and recommended action
* block “Continue” to Step 2 if any **errors**
* allow continue with warnings but require checkbox “I confirm these numbers are correct”
