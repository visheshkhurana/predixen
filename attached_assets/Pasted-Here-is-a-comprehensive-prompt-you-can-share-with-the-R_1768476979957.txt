## 🛠️ Detailed Bug & Improvement Report for RunwayAI

### 1. **Onboarding — Financial Baseline**

**Reproduction Steps:**

1. Navigate to the onboarding flow and fill out *Company Info* with a name, website and select **Series A** stage.
2. On the **Financial Baseline** screen, leave default values or adjust as needed.
3. Click **Run First Truth Scan**.

**Actual Result:**
A toast appears with “Request failed”; the workflow does not progress to Step 3. Repeated clicks still fail.

**Expected Result:**
Upon submission, a truth scan should run, persist data, and move the user to the final onboarding step, enabling the rest of the app.

**Possible Fixes:**

* Inspect backend API call associated with `button-run-scan`; check for network errors or missing credentials.
* Ensure the API endpoint accepts demo data or simulate values when a real accounting integration isn’t connected.
* Add error handling to surface meaningful messages (e.g., “Service unavailable” or “Missing API key”) rather than a generic failure.

### 2. **Truth Scan Page**

**Observation:** Once bypassing onboarding, the page loads with placeholders for 24 metrics, yet all show “N/A” and both scores are `0/100`.

**Expected Behavior:**
If no scan has been run, show a clear call‑to‑action instructing the user to run the scan. If a scan is in progress, display a loading indicator or last known results.

**Recommendations:**

* Use conditional rendering based on scan status (no scan, pending, success, failure).
* Consider populating demo users with sample metrics to give an idea of what a successful scan looks like.

### 3. **Simulation (Scenario Builder)**

**Issues Encountered:**

* Tutorial overlay needs to be manually closed; otherwise, UI elements are blocked.
* Selecting templates (e.g., “Baseline” or “Conservative Cut”) doesn’t progress to subsequent wizard steps. Estimated runway remains static (16.7 months).

**Expected Behavior:**
Selecting a template should update state and allow Step 2 (Parameters) to be opened. Changing presets should modify the summary panel (pricing, growth, burn reduction, etc.).

**Suggestions:**

* Ensure that a scenario template selection triggers state changes; check event handlers attached to template buttons.
* Provide an obvious **Skip / Close Tutorial** option for returning users.
* Implement loading or disabled states if the truth scan prerequisites are unmet, with instructions to complete them first.

### 4. **Templates Page**

**Observation:** The “Scenario Templates” section shows “Loading templates…” indefinitely.

**Expected Behavior:**
Display a searchable list of templates with descriptions and tags.

**Recommendations:**

* Verify that the templates endpoint is called and that responses are correctly parsed.
* For demo accounts, either provide a small set of templates or hide this page until data is available.

### 5. **Decisions Page**

**Observation:** After pressing **Regenerate Decisions**, the page displays “No Decisions Yet. Run a simulation to generate ranked recommendations.”

**Expected Behavior:**
Once a simulation is run, this page should rank top actions by expected impact.

**Suggestions:**

* Ensure that decision generation only depends on simulation outputs; if there’s no simulation, show guidance rather than a dead end.
* Provide a link or button to take the user back to the Simulation page to start a scenario.

### 6. **Copilot**

**Issue:** Copilot surfaces an error toast reading “Run a truth scan first” whenever the page is loaded.

**Expected Behavior:**
The Copilot should either:

* Be disabled with a clear explanation until a scan is complete, or
* Be able to answer generic questions with placeholder/dummy data when no real metrics are available.

**Improvement:**
Implement a fallback mode that provides sample responses, explaining that richer insights require connecting data and running a truth scan.

### 7. **Alerts & Monitoring**

**Issue:** The page shows an error toast “Run a truth scan first,” and no alerts or driver health data.

**Expected Behavior:**
Alerts and driver health should update when underlying metrics change or thresholds are breached.

**Recommendations:**

* Hide or disable these tabs until dependencies (truth scan + simulation) are complete.
* Provide informative onboarding explaining why alerts are empty.

### 8. **Integrations Page**

**Observation:** This page loads correctly and simply states that Accounting and CRM are “Not connected.”

**Improvements:**

* Offer direct calls‑to‑action to connect QuickBooks, Xero, HubSpot, etc.
* Add tooltips explaining benefits of integration and possible demo modes.

### 9. **Overall Product Suggestion**

* Implement a “Demo Mode” for new users that populates the application with sample data, allowing them to explore features without needing to complete all integrations or run real scans.
* Improve error handling across the app; instead of generic toasts, offer next steps and documentation links.
* Consider progressive feature gating: when a prerequisite (like a truth scan) is missing, show a banner on dependent pages with a clear “Complete this step” button.
