### 🛠 Replit Prompt to Improve the Scenario Builder UI

#### Context

You’re working on a startup financial simulator called **RunwayAI**. The current Scenario Builder screen has a crowded layout with multiple sliders, dropdowns, and template panels all visible at once. Users find it overwhelming and hard to navigate. Your task is to refactor this UI to be more intuitive, accessible, and visually balanced.

#### Key Objectives

1. **Reduce Cognitive Load**: Guide users step-by-step instead of exposing every control simultaneously.
2. **Improve Layout and Hierarchy**: Group related elements, align them neatly, and use clear headings.
3. **Enhance Learnability**: Provide tooltips or helper text for financial sliders and tags.
4. **Offer Interactive Feedback**: Show real-time or preview data when adjusting inputs.
5. **Ensure Accessibility**: Support keyboard navigation and high-contrast modes.

#### Required Changes

1. **Implement a Step Wizard**

   * Break the scenario configuration into stages:

     1. **Template Selection**: Show a simplified list of templates with concise descriptions (e.g., baseline, conservative cut, growth push).
     2. **Parameter Adjustment**: Present sliders for pricing, growth, burn reduction, etc., grouped together.
     3. **Fundraising Setup**: Ask for fundraising month and amount.
     4. **Review & Run**: Summarize the chosen settings and provide a “Run Simulation” button.
   * Use a progress bar or numbered steps to indicate user position in the workflow.

2. **Improve Layout**

   * **Group Controls**: Place all financial sliders in one section with a single heading (e.g., “Adjust Financial Parameters”). Similarly, place tags and scenario name input in a dedicated area.
   * **Consistent Alignment**: Ensure labels and input elements line up vertically to improve readability.
   * **Spacing**: Add uniform margins/padding between sections to prevent clutter.

3. **Add Helper Text and Tooltips**

   * For each slider (Pricing Change, Growth Uplift, Burn Reduction, Gross Margin Change), implement a small “ℹ️” icon that shows an explanation of what the slider does and the expected impact on runway length or profitability.
   * Display current value alongside the slider in real time (e.g., “Pricing Change: 5% → moderate”).

4. **Real-Time Feedback**

   * When a user moves a slider, display a dynamic summary (e.g., “Burn Reduction: –15% → extends runway by 3 months”) without requiring the full simulation to run. This can be a small text element beneath each slider that updates as the value changes.

5. **Accessibility Improvements**

   * Ensure all inputs are reachable via `Tab`/`Shift+Tab`, with visible focus states.
   * Provide a high-contrast theme toggle (dark/light or accessible mode).
   * Use ARIA labels for buttons, sliders, and form fields.

6. **Navigation Enhancements**

   * Make the side navigation collapsible, so it shrinks into icons when users need more space.
   * Implement breadcrumb links or sub-tab navigation at the top to distinguish between “Scenario Builder” and “Simulation Results”.

#### Technical Considerations

* The project is likely built with React or another front-end framework. Structure each step of the wizard as a separate component or route.
* Use state management (e.g., React’s `useState` or Context API) to preserve user inputs between steps.
* Make sure to test for responsiveness; the UI should adapt well on smaller screens.

#### Deliverables

* Refactored Scenario Builder UI with the step-by-step flow and grouped layout.
* Updated components implementing tooltips and real-time feedback.
* Accessibility enhancements and keyboard-friendly navigation.
* Clean, maintainable code hosted on Replit, ready to test in the existing project environment.
