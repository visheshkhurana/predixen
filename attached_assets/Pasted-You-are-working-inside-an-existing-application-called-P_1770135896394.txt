You are working inside an existing application called **Predixen**.

Predixen already has (or is implementing):
- A Data Source Integration backend
- Connector adapters (Stripe, QuickBooks, GA4, REST API, DB, CSV, etc.)
- Org-based multi-tenancy
- Ingestion runs, raw_events, metric catalog

Your task is to build a **Connector Marketplace UI** similar to Klipfolio’s integration gallery.

GOALS:
1. Help users DISCOVER available integrations easily
2. Clearly show which connectors are:
   - Native
   - Generic
   - Beta
3. Make setup fast and guided (wizard-based)
4. Show trust indicators (data freshness, reliability)
5. Be scalable to 50–100+ connectors without redesign

DESIGN PRINCIPLES:
- Marketplace feel (cards, categories, search)
- Zero ambiguity on what each connector does
- Enterprise-safe (no accidental auth leaks)
- Optimized for SMB onboarding speed

---

## 1️⃣ DATA MODEL — CONNECTOR REGISTRY (BACKEND)

Create a static + extensible **Connector Registry**.

Create file:
server/modules/integrations/connectorRegistry.ts

Structure:
- id (string, stable)
- name (Stripe, Google Analytics, PostgreSQL, REST API, etc.)
- category (Finance, Marketing, CRM, Databases, Files, Custom)
- logo_url (or local asset ref)
- description (short)
- long_description (markdown)
- auth_type (oauth, api_key, webhook, db_connection, file_upload)
- supports_webhooks (boolean)
- supports_polling (boolean)
- supports_incremental (boolean)
- typical_refresh (e.g. "real-time", "hourly", "daily")
- native (boolean)
- beta (boolean)
- popularity_rank (int)
- setup_complexity (low | medium | high)
- documentation_url
- implemented (boolean)
- adapter_key (maps to backend adapter)

Expose API:
GET /api/connectors/catalog

This MUST be the single source of truth for the marketplace UI.

---

## 2️⃣ BACKEND — CONNECTOR MARKETPLACE API

Create routes:
GET /api/connectors/catalog
GET /api/connectors/catalog/:id

Response must include:
- connector metadata
- whether org already has this connector configured
- current status if installed (active, error, paused)
- last successful sync timestamp
- error summary (if any)

DO NOT expose secrets.

## 6️⃣ SETUP WIZARD INTEGRATION

When user clicks "Connect":

Route to:
client/pages/AddDataSourceWizard.tsx

Wizard behavior must depend on connector.auth_type:
- oauth → redirect flow
- api_key → secure input
- db_connection → connection string form
- webhook → show endpoint + secret
- file_upload → upload UI

Wizard steps:
1. Auth
2. Test connection
3. Select entities/streams (if applicable)
4. Choose sync mode (webhook/poll)
5. Choose refresh frequency
6. Confirm & activate

On completion:
- Create data_source record
- Trigger first ingestion run
- Redirect to DataSourceDetail page
