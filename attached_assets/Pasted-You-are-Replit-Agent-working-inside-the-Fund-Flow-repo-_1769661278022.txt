# 2) Truth Scan computation: use existing data first

## Input sources

A) ImportSession:

* Use FinancialMetricPoint rows as the primary source of time series (period + metric_key + value/value_raw).
* Optionally use FinancialRecord rows for cross-checks (period_start/end aggregates).

B) Dataset:

* For financial dataset type: pull from FinancialRecord rows produced by CSV ingestion.
* For transactions/customers: only used to compute optional metrics (concentration, churn proxies) later.

C) Manual baseline:

* treat as one “period” (selected_period or current month), with low confidence unless user confirms.

## Canonical mapping

Map metric_key -> canonical:

* revenue -> revenue
* cogs -> cogs
* cash_balance -> cash_balance
* payroll -> opex_by_category.payroll
* marketing_expense -> opex_by_category.marketing
* other_costs -> opex_by_category.other_costs
* opex (if present) -> opex_total
  If both opex_total and categories exist, validate reconciliation.

Use ImportSession.detected_time_granularity and detected_sign_convention as priors for assumptions, but still confirm if ambiguous.

# 3) Services: server/services/truth_scan.py

Implement a deterministic pipeline:

1. create_truth_scan_upload(company_id, source_kind, import_session_id|dataset_id|baseline_payload)
2. infer_assumptions(upload):

   * currency: from FinancialMetricPoint.unit if consistent; else company default; else NEEDS_USER (BLOCKED)
   * scale: infer from labels if available in raw_data/extraction_summary; else assume "unit" with MEDIUM confidence and ask if values look 100x/1000x off
   * time_granularity: from ImportSession.detected_time_granularity; else infer from period spacing
3. build_truth_dataset(upload):

   * Build monthly facts JSON with provenance:

     * For each month+metric, store sources referencing metric_point ids (and record ids if used)
     * confidence: map "high/medium/low" -> float (0.9/0.6/0.3)
     * if value_raw exists and differs, store derivation note
4. rule_validate(truth_dataset):
   Implement the exact tiers:

TIER 1 STRUCTURAL (BLOCKED)

* missing currency OR time_granularity
* no time series for revenue OR (opex_total or payroll/other_costs) OR cash_balance
* dates not parseable to YYYY-MM

TIER 2 ARITHMETIC/ACCOUNTING (HIGH)

* Revenue=0 for all months but expenses>0 AND cash deltas inconsistent
* Burn sign mismatch (profitable month but net_burn negative labeling)
* Runway computed when burn<=0: must be “Cash-generating”
* Net burn formula per month:
  burn = (opex_total + cogs - revenue)
  If burn < 0 => label as profit and set runway="Cash-generating"

TIER 3 PLAUSIBILITY (MEDIUM)

* growth==0 for all months while revenue varies
* concentration N/A when customer breakdown exists
* churn==0 when subscription metrics exist

TIER 4 COMPLETENESS (LOW)

* optional metrics missing

CONFLICT

* if multiple sources disagree for same month+metric beyond tolerance, create CONFLICT issue with both evidences.

5. safe_repair(truth_dataset, issues):
   Auto-fix ONLY:

* derive net_burn_monthly using formula
* runway logic with burn<=0 => “Cash-generating”
* growth calculations, but if divide-by-zero or missing prev month => “insufficient data” not 0
* fill N/A if inputs exist
  Log truth_decision_logs(action=AUTO_FIX_APPLIED) with patch.

6. llm_assist (optional, gated):
   Use existing LLM router but only for:

* ambiguous scale/currency/timebase detection from ImportSession.raw_data or extraction_summary
* phrasing user questions + option buttons
  NEVER invent numbers. Output JSON only.

7. create_validation_report(truth_dataset, issues, decisions)
   Persist validation_reports + validation_issues.

8. finalize_truth_dataset(upload):
   Allowed only if no OPEN BLOCKED issues.

* mark truth_dataset.finalized=true, is_latest swap for company
* set companies.latest_truth_dataset_id
* update ImportSession.status="truth_scan" then "saved" when finalized (keep flow explicit)
* log truth_decision_logs(DATASET_FINALIZED)

1. POST /truth-scan/from-import-session
   Body: {company_id:int, import_session_id:int}

* Builds truth dataset + report
* Returns {upload_id, truth_dataset_id, report_id, status, issues_summary}

2. POST /truth-scan/from-dataset
   Body: {company_id:int, dataset_id:int}

* same return

3. POST /truth-scan/from-manual-baseline
   Body: {company_id:int, baseline_payload:object}

* same return

4. GET /truth-scan/uploads/{upload_id}
   Return:

* assumptions + coverage + confidence summary
* key metrics snapshot latest month
* issues grouped: auto_fixed, needs_confirmation, blocked
* questions[] (max 7) generated from unresolved issues

5. POST /truth-scan/uploads/{upload_id}/resolve
   Body: {answers:[{metric_key, value|choice, notes?}]}

* Apply override patch to truth_dataset.assumptions/facts (do not mutate original metric_points)
* Log USER_OVERRIDE_APPLIED
* Re-run validate + safe_repair + update report/issues
* Return updated truth scan

6. POST /truth-scan/uploads/{upload_id}/finalize

* Enforce no BLOCKED issues open
* finalize dataset and set company.latest_truth_dataset_id

7. GET /companies/{company_id}/truth/latest

* Return latest finalized truth dataset for simulator + copilot
