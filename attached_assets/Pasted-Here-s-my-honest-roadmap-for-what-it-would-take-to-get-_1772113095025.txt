# The 10/10 Roadmap for FounderConsole.ai

## Phase 1: "Make It Work" (P0 — Current Score: 5.6 → 7.5)
*Timeline: 4–6 weeks*

### 1. Fix Simulation Reliability
- **422 errors on custom queries** must be zero. Every natural-language scenario a founder types should parse and simulate successfully.
- "Run Another Scenario" button must actually work — this is the core loop.
- Baseline runway in simulator must match dashboard runway exactly (21.8 months ≠ 120 months is unacceptable).

### 2. Fix KPI Consistency
- Burn rate cannot show $70K, $82.6K, and $107K in the same session. One source of truth, everywhere.
- Every KPI needs a clickable audit trail: **Source → Formula → Assumptions → Confidence**
- Synthetic/estimated KPIs (CAC, LTV, Churn, NRR, ARPU) that appear without user data must be labeled **"AI-Estimated"** with visible assumptions, not "Verified."

### 3. Add Stripe Integration
- This is non-negotiable for SaaS. No Stripe = no trust in revenue data = no payment.
- Must pull: MRR, churn, ARPU, expansion/contraction revenue, trial conversions
- Data should appear within 5 minutes of connection, not hours.

## Phase 2: "Make It Trustworthy" (P1 — Score: 7.5 → 8.5)
*Timeline: 6–10 weeks*

### 4. Simulation Sanity Checks
- Add mathematical validation: if inputs say "CAC +20%" the output cannot show "Capital Efficiency 10/10"
- Add a visible **"Math Check"** panel: *"Starting cash $1.8M ÷ Modified burn $X/mo = Y months"*
- Flag contradictory results automatically: *"⚠️ This adverse scenario shows improved metrics — review assumptions"*
- Scenario comparison table: side-by-side view of all simulated scenarios on one screen

### 5. Data Confidence Score 2.0
- Break confidence into components: Revenue (80%), Expenses (50%), Growth (30%), Retention (20%)
- Show actionable next steps: *"Connect Stripe to increase confidence from 62% → 85%"*
- Add staleness indicators: *"Revenue data is 14 days old — refresh recommended"*
- Distinguish between "Verified" (user-confirmed), "Connected" (from integration), and "Estimated" (AI-generated)

### 6. Add HubSpot + Chargebee/Recurly Integrations
- HubSpot: pipeline value, deal velocity, lead-to-customer conversion
- Chargebee/Recurly: subscription metrics, plan distribution, upgrade/downgrade rates
- Each integration needs: connection time estimate, permission scope explanation, reconnect flow, error recovery UI

### 7. Decision Journal v1
- Save every simulation with timestamp, inputs, outputs, and founder notes
- Add status labels: "Executed," "Deferred," "Abandoned"
- Monthly email prompt: *"You simulated cutting marketing 25% on Feb 26. What happened?"*
- Prediction vs. reality tracking with variance visualization
