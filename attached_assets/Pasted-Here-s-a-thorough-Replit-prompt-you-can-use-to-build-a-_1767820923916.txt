**Prompt for Replit AI:**

> **Project goal**
> Build a full‑stack application where users can upload a Termina report (PDF). The backend should extract all relevant financial/operational metrics from the uploaded PDF and call the OpenAI API to analyse/summarise the content. The extracted metrics should then be stored as baseline data for that company, and the app should include a simple simulation engine that lets users apply parameter changes (e.g., adjust growth rates or margins) to see projected outcomes.
>
> **Desired stack**
> • Use Python with Flask for the backend.
> • Use a lightweight front‑end (could be plain HTML/CSS/JS or React) served by Flask.
> • Use SQLite for storing extracted metrics and simulation scenarios.
>
> **Functional requirements**
>
> 1. **File Upload:**
>       – Create an upload page where a user can select a PDF file (Termina Mini Scan).
>       – Validate that only PDFs are accepted.
>       – Store the uploaded file on the server in an `/uploads` directory.
> 2. **PDF Parsing & Text Extraction:**
>       – Use `pdfplumber` (or a similar library) to extract text from the uploaded PDF.
>       – If the PDF includes images rather than text, gracefully handle extraction errors and inform the user.
> 3. **OpenAI API Integration:**
>       – Use the extracted text as input to the OpenAI API (e.g., `gpt-3.5-turbo` or `gpt-4`), asking it to summarise the report and identify key financial metrics (e.g., monthly revenue, gross margin, operating income, growth rates).
>       – The API key should be stored securely using environment variables (e.g., `os.environ["OPENAI_API_KEY"]`).
>       – Define a clear prompt for OpenAI, such as: “Extract from the following text the company’s revenue for the most recent month, run‑rate revenue, COGS, gross profit, gross margin, operating income, operating margin, YoY growth, and any other performance metrics. Return the result as a JSON object with appropriate keys.”
> 4. **Data Persistence:**
>       – After parsing the OpenAI response, save the metrics to a SQLite database table (`companies`), keyed by company name and report date.
>       – Also store the raw OpenAI summary for future reference.
> 5. **Baseline Metrics Display:**
>       – Provide a dashboard page that shows the stored metrics for each company.
>       – Allow users to click on a company to view its baseline metrics (e.g., revenue, margins) and the date they were extracted.
> 6. **Simulation Engine:**
>       – Build a simple module where a user can adjust parameters like revenue growth rate, gross margin, and operating margin.
>       – Use the baseline metrics as defaults. When parameters are changed, calculate new projected revenue, gross profit and operating income based on the user’s adjustments.
>       – Display simulation outputs in a table or chart (e.g., using Chart.js or simple HTML tables).
> 7. **Error Handling & UX:**
>       – Provide clear error messages if PDF extraction fails or the OpenAI API call returns an error.
>       – Show a progress indicator while the PDF is being processed and analysed.
>
> **Implementation hints**
> • In `app.py`, define routes for uploading files (`/upload`), processing them (`/process`), and viewing dashboard/simulations.
> • Create a `templates` folder with Jinja2 templates for upload, dashboard, and simulation pages.
> • Write a helper function `parse_pdf(file_path)` that uses pdfplumber to extract text and returns it as a single string.
> • Write a helper function `call_openai(text)` that calls the OpenAI API with your extraction prompt and returns a parsed JSON dict of metrics.
> • Store metrics in SQLite with fields: company_name, report_date, revenue, run_rate_revenue, cogs, gross_profit, gross_margin, operating_income, operating_margin, yoy_growth, extracted_at, summary_text.
> • For the simulation module, read the latest metrics from the database and multiply values by the user‑specified adjustment factors. Display updated numbers without altering the stored baseline.
>
> **Testing**
> • Include a sample PDF in the repo (or use the user‑uploaded file path) to verify extraction.
> • Write unit tests to ensure that the PDF parser returns text and that the OpenAI call returns correctly structured JSON.
>
> **Deployment considerations**
> • Use Replit’s secrets management to store your OpenAI API key.
> • Ensure that the `uploads` directory is included in `.gitignore` to avoid committing user files.
> • Provide instructions in a `README.md` on how to set up, run the application, and obtain an OpenAI API key.
