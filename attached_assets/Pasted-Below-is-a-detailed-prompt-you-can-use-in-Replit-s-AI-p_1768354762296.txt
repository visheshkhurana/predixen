**Replit Prompt – Enhance Scenario Builder UX and Parameter Inputs**

I am working on a startup financial simulation tool in a React/Next.js project (currently deployed at `/scenarios` on our app). The interface contains a multi-step “Scenario Builder” wizard with sliders for **Pricing Change**, **Growth Uplift**, and **Burn Reduction**, and includes steps for Template, Parameters, Fundraising, and Review. I’d like to significantly enhance this tool’s user experience and capture more nuanced input from users. Please implement the following improvements:

### 1. Guided Onboarding

* Add a first-time user tutorial or walkthrough that highlights each step of the wizard.
* Provide a brief description of what information is expected at each step (e.g. Template selection, adjusting financial levers, modelling fundraising rounds, and reviewing the final scenario).
* Ensure the tutorial can be skipped or revisited via a “Help” or “Tour” button.

### 2. Enhanced Interactivity and Feedback

* For each slider (Pricing Change, Growth Uplift, Burn Reduction), include tooltips that describe how changes affect revenue, growth or cost structure. Use simple examples (e.g. “Increasing pricing by 10% raises ARPU by 10%”).
* Implement real‑time updates of key metrics like **Estimated Runway**, cash-on-hand, and projected revenue/loss as the user adjusts sliders. Display these metrics in a small dashboard or sidebar within the wizard.

### 3. Scenario Management and Comparisons

* Allow users to save scenarios with custom names, and display a list of saved scenarios for easy retrieval and comparison.
* Implement version control: track changes to a scenario over time and allow users to revert to previous versions.
* Enable side‑by‑side comparison of multiple scenarios (e.g. “Base Case”, “Optimistic”, “Worst Case”) and integrate this into the existing “Compare All” tab.

### 4. Accessibility Improvements

* Ensure all interactive elements (sliders, buttons, tabs) are keyboard navigable.
* Provide optional text input fields alongside sliders so users can type exact values.
* Add high‑contrast mode and ensure tooltips and labels are screen‑reader friendly.

### 5. Visual Timeline and Context Sidebar

* Create a timeline view where users can place major events (product launches, funding rounds, growth initiatives) and drag them along the timeline to change their timing.
* Add a persistent summary sidebar that shows the user’s current assumptions and inputs across the wizard steps, updating live as they make changes.

### 6. Expanded Parameter Inputs

* **Financial Baseline**: Ask the user for current monthly burn rate, current runway (in months), and current ARR/MRR.
* **Revenue Model**: Gather assumptions for average revenue per user (ARPU), churn rate, customer acquisition rate, and any seasonality factors.
* **Growth Drivers**: Include inputs for marketing spend, projected sales pipeline growth, and planned product or market expansions.
* **Cost Structure Breakdown**: Allow users to adjust burn reduction by category (payroll, marketing, R&D, infrastructure, G&A) rather than a single percentage slider.
* **Customer Segmentation**: Offer the ability to define multiple customer segments with distinct ARPU, churn, and growth rates.
* **Pricing Tiers**: If applicable, ask for conversion rates between pricing tiers and model how tier migration affects total revenue.
* **Fundraising Details**: Collect information about planned fundraising amount, target close date, expected valuation, and investment terms (convertible note vs priced round, interest rates, maturity dates).
* **Runway Sensitivity Analysis**: Let users specify minimum and maximum runway targets, and generate scenario outputs showing best‑ and worst‑case runway lengths based on varying revenue and cost assumptions.
* **Macro‑Economic Factors**: Provide optional sliders or dropdowns for inflation, market growth rate, or competitive pressure that apply a global adjustment to revenue and cost projections.
* **Exit Strategy**: Ask users about desired exit timeline and revenue multiple targets (e.g. “exit in 3 years at 5× revenue”) and project whether their current plan aligns with these goals.

### 7. Validation and User Guidance

* Implement validation to flag unrealistic inputs (e.g. 300% growth uplift on a small base) and offer gentle guidance or default ranges.
* If users enter extreme or conflicting data, prompt them to revise the input or provide context for why such an assumption is reasonable.
