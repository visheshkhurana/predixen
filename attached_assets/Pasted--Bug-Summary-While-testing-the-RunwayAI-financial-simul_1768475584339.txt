**Bug Summary**
While testing the RunwayAI financial simulator on Replit with test data, I encountered multiple blocking errors that prevented completing the onboarding, saving data, running truth scans, or using the scenario builder.

**Steps to Reproduce**
1. Launch the app and complete the onboarding Company Info step with any values.
2. In the Financial Baseline step, enter values for revenue, gross margin, opex, payroll, other costs and cash balance.  Click **Run First Truth Scan**.
   - Expected: Should run a scan and move on.
   - Actual: Shows toast “Request failed” and stays on the same step.
3. Go to **Data Input > Company**.  Fill out founding date, stage, industry, description and number of employees.  Click **Save All Data**.
   - Expected: Data saved successfully.
   - Actual: Toast error “Failed to save – There was an error saving your data”.
4. On **Data Input > Financials**, fill in cash on hand, monthly revenue, monthly expenses, growth rate and optional expense breakdown.  Observe computed metrics.
   - Issue: Computed totals become astronomically large (e.g., net burn and total expenses), indicating a calculation bug (string concatenation instead of numeric addition).
5. On **Data Input > Goals**, set target runway, growth scenario, funding target and timeline, then attempt to save.
   - Issue: “Save All Data” button sometimes missing; no confirmation of save; likely same backend failure.
6. Navigate to **Truth Scan** and click **Refresh Scan**.
   - Issue: All key metrics display “N/A” and Data Confidence Score remains 0/100, reflecting no data processed.
7. Navigate to **Simulation**.  Close tutorial overlay, select any template (Baseline, Conservative Cut, etc.).
   - Issue: Template selection does not progress to Step 2 (Parameters).  The simulation wizard stays stuck at Step 1.

**Observed Error Messages**
- “Request failed” when running truth scan.
- “Failed to save – There was an error saving your data” when saving company/financial data.
- Metrics show “N/A” in Truth Scan.
- Unrealistic computed numbers on Financials page.

**Environment**
- Hosted on Replit at `https://fund-flow--vysheshk.replit.app`
- Tested on Chromium.

**Hypotheses**
- Backend endpoints (save, scan, scenario creation) are failing or unreachable (CORS or network error).
- Numeric values may be treated as strings when computing totals.
- Application state is not persisted when save operations fail.

**Request**
- Check server/API logs to see why `/api/save` and `/api/truth-scan` requests fail.
- Verify that environment variables and database connections are configured correctly.
- Ensure numeric inputs are parsed as numbers in calculations.
- Confirm that scenario builder can proceed once baseline data is saved successfully.
