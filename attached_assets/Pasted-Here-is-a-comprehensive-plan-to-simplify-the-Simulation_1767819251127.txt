## 🧠 Suggested Backend Algorithm

Below is a high‑level algorithm for generating five common scenarios. Adapt it to your existing data structures.

1. **Define Scenario Parameters**:

   * **Baseline**: Current growth and burn rates, no fundraising.
   * **Conservative Cut**: Lower growth, reduced expenses (burn reduction).
   * **Moderate Growth**: Slightly higher growth and modest burn reduction.
   * **Aggressive Growth**: High growth uplift, possibly increased burn and pricing uplift.
   * **Cost‑Cutting**: Significant burn reduction, minimal growth.

2. **Parameter Distributions**:

   * For each scenario, specify probability distributions for key variables (pricing change %, growth uplift %, burn reduction %, gross margin %, fundraising month and amount).
   * Example: Baseline growth uplift ~ Normal(0%, σ=1%), aggressive growth uplift ~ Normal(10%, σ=3%), burn reduction ~ Normal(−5%, σ=2%), etc.

3. **Monte Carlo Simulation**:

   * For each scenario, perform N simulations (e.g., 1 000) over the forecast horizon (e.g., 24 months).
   * For each simulation:

     1. Sample the key variables from their distributions.
     2. Calculate revenue, costs, and cash flow for each month based on the sampled parameters and current financial model.
     3. Update cash balance month by month.
   * Collect results to compute percentiles (P10, P50, P90) for each month.

4. **Output Structure**:

   * Produce a data structure that holds month‑indexed metrics for each scenario. For example:

     ```json
     {
       "Baseline": {
         "month": [1, 2, 3, ...],
         "revenue_p50": [...],
         "revenue_p10": [...],
         "revenue_p90": [...],
         "cash_p50": [...],
         "burn_p50": [...]
       },
       ...
     }
     ```

5. **Selecting “Most Common” Scenarios**:

   * If user history or data indicates frequently used strategies, prioritize those. Otherwise use the five typical scenarios above.
   * Optionally allow users to customize which scenarios are generated.

6. **Backend API Endpoint**:

   * Create an endpoint (e.g., `POST /simulate`) that accepts scenario definitions and returns the month‑indexed simulation results.
