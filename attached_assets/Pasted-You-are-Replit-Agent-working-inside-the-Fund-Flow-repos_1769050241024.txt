========================================================
A) FUNDRAISING DATA MODEL (ROUND + SECURITY + INVESTOR)
=======================================================

Goal: First-class fundraising objects that power modeling and docs.

---

## A1) Tables

1. `company_cap_tables`

* id (uuid)
* company_id (fk)
* name (text) // e.g. "Current Cap Table"
* as_of_date (date)
* currency (default USD)
* cap_table_json (json) // canonical structure (see below)
* created_at, updated_at

Canonical `cap_table_json` minimum:
{
"common": [{ "holder": "Founder A", "shares": 0, "percent": 0 }],
"preferred": [{ "series": "Seed", "holder": "Fund X", "shares": 0, "liquidation_pref": "1x non-participating" }],
"options": { "pool_percent": 0.0, "allocated_percent": 0.0 },
"notes": [{ "holder": "", "principal": 0, "conversion_cap": 0, "discount": 0.0 }],
"fully_diluted_shares": 0
}

2. `fundraising_rounds`

* id
* company_id
* name (text) // Seed / A / B / Bridge
* target_raise (numeric)
* pre_money (numeric)
* post_money (numeric)
* instrument (enum: equity|safe|note)
* option_pool_refresh_percent (numeric nullable)
* use_of_funds_json (json)
* status (enum: planned|active|closed)
* created_at, updated_at

3. `round_terms`

* id
* round_id
* terms_json (json) // see term sheet model below
* created_at, updated_at

4. `investors`

* id
* company_id
* name
* type (enum: vc|pe|angel|strategic|family_office)
* geography
* stage_focus (text)
* thesis_tags (json array)
* contact_json (json) // optional
* created_at

5. `investor_pipeline`

* id
* round_id
* investor_id
* stage (enum: sourced|contacted|meeting|dd|term_sheet|committed|passed)
* probability (0–1)
* last_contacted_at
* notes (text)
* created_at, updated_at

---

## A2) Term Sheet JSON model (MVP)

terms_json should support:

* valuation: pre_money, post_money
* security: preferred/common/safe/note
* liquidation_preference: multiple + participating + cap
* dividends: rate + cumulative
* anti_dilution: broad-based weighted average / full ratchet / none
* pro_rata_rights: yes/no + threshold
* board: composition
* protective_provisions: list
* option_pool: size + refresh
* vesting: founder vesting reset? yes/no
* closing_conditions: list
* legal: governing law, counsel, fees
