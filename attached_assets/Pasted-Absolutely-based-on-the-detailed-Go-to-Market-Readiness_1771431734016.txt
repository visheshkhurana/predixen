🛠 Fix Bundle 5: NLP Intent Correct Mapping

Cause
Parser rules not mapping negative verbs correctly.

Fix
In server/copilot/intent_parser.py:
	•	add patterns for “cut|reduce|decrease” → negative operator
	•	add patterns for “increase|raise|add” → positive operator

Patch

NEGATIVE_WORDS = ["cut","reduce","decrease","drop"]
POSITIVE_WORDS = ["increase","add","raise","grow"]

def parse_change_intent(text):
    for w in NEGATIVE_WORDS:
        if w in text:
            return {"direction": -1}
    for w in POSITIVE_WORDS:
        if w in text:
            return {"direction": 1}
    return {"direction": 0}

Tests
	•	“Cut marketing 25%” → negative
	•	“Increase revenue 15%” → positive
	•	“Reduce burn” → negative
