Project Title: Onboarding PDF Fixes and Truth Scan Enhancements

Context:
When a user uploads a pitch deck (PDF) in the onboarding flow, data extraction works but clicking “Continue” does nothing, whereas loading sample data correctly proceeds to Step 2.  On the Truth Scan page, certain metrics (burn multiple, revenue growth) show as N/A or 0, and the page lacks a business summary.  Insights and recommendations are currently thin.

Goals:
1. Fix onboarding progression after PDF upload.
2. Ensure burn multiple and revenue growth calculations populate correctly.
3. Add a business summary section at the top of the Truth Scan results.
4. Provide an option to fetch missing summary information from online sources when not in the deck.
5. Make insights and recommendations more detailed and actionable.

### 1. Fix Onboarding “Continue” after PDF Upload

- **Investigate the Onboarding Form Submission**
  - Locate the component handling the PDF upload (e.g., `DropzoneDeckUpload`) and the subsequent form submission.
  - The `onSubmit` handler for the “Continue” button should parse the extracted PDF data and populate the company info (name, industry, stage) as if they were entered manually.
  - Ensure that once data is extracted, `nextStep()` or the step‑index state change is triggered when the user clicks Continue.
  - If the PDF extraction returns an error or incomplete data, display an inline error and enable manual entry rather than silently blocking progression.

- **Add Loading and Validation States**
  - While the extraction is processing, disable the Continue button and display a spinner.
  - After extraction, validate that required fields (company name, industry, stage) are filled—either by extracted data or user input—and only then allow Continue.
  - If extraction yields partial data, prompt the user to complete missing fields before proceeding.

### 2. Correct Burn Multiple and Revenue Growth Calculations

- **Backend Calculation Logic**
  - Audit the calculation functions for burn multiple and revenue growth:
    - **Burn Multiple** should be computed as Net Burn divided by Net New ARR.  Net Burn is typically cash out minus cash in over a period; Net New ARR is the change in annual recurring revenue during the same period.  Ensure these inputs are pulled correctly from the user’s data or baseline values.
    - **Revenue Growth** should be `(current period revenue – previous period revenue) / previous period revenue`.  For the first truth scan, compare monthly revenue against the baseline month extracted from the deck or user input.
  - If either numerator or denominator is missing, display a tooltip explaining why (e.g., “Need at least two months of revenue data to calculate growth”).  Only show “N/A” when truly not computable.

- **API Updates**
  - If calculations are performed on the backend, update the API endpoints to return burn multiple and revenue growth values in the results JSON.
  - Store the raw values and the formatted display strings (e.g., `1.2x` for burn multiple, `15%` for revenue growth) and send them to the front‑end.

- **Front‑End Rendering**
  - On the Truth Scan page, remove hard‑coded defaults for these metrics.  Bind the UI fields to the actual values returned from the API.
  - When a metric is missing, display a clear note (“Not enough data to compute burn multiple”) with a link to the Data Input page.

### 3. Add Business Summary to Truth Scan Page

- **Deck-Based Summary Extraction**
  - If the uploaded pitch deck contains a company overview or executive summary slide, extract up to ~100 words using your existing PDF extraction service.
  - Save the summary in the scan results and display it at the top of the Truth Scan page under a heading like “Business Overview”.

- **Fallback Summary Retrieval**
  - If no summary is found in the deck:
    - Provide an option for users to fetch a summary from online sources (e.g., the company’s website or Crunchbase/LinkedIn).  Implement a button “Fetch Company Summary” that:
      - Makes an API call to a scraping or summarisation service (or a pre‑integrated third‑party API) to gather a short description of the company.
      - Populates the business summary section with the fetched text.
    - Explain the source of the summary (“Pulled from public profiles”) and allow users to edit or approve it before saving.

- **Editable Summary Field**
  - Make the summary field editable so users can refine or replace it with their own text.
  - Save the final summary in the database for future scans.

### 4. Detailed Insights and Recommendations

- **Insights Generation Logic**
  - Improve the recommendation engine to use more variables (burn multiple, revenue growth, gross margin, top‑5 concentration, runway, cash balance).  For example:
    - If burn multiple > 2, suggest reducing operating expenses or improving revenue growth.
    - If revenue growth is low but gross margin is high, recommend investing in marketing or sales.
    - If runway < 6 months, advise raising capital or cutting burn.
  - Include at least three distinct, actionable recommendations based on the metrics.

- **Richer Insight Text**
  - Instead of generic messages like “Complete your data”, give context: “Your burn multiple is high because net new ARR is negative—explore ways to increase ARR or cut burn to below $X/month.”
  - Use bullet points or numbered lists for readability.

- **Front‑End Display**
  - On the Truth Scan page, display the recommendations under a heading like “Key Insights” with icons or status indicators (e.g., warning, info).
  - If no insights apply, show a supportive message like “You’re on track! Keep monitoring your metrics regularly.”

### Deliverables

1. **Bug fix** for the PDF onboarding flow so “Continue” moves to Step 2 after extraction.
2. **Corrected formulas and API responses** for burn multiple and revenue growth.
3. **Business summary section** on Truth Scan, populated from the deck or fetchable via external sources and editable by the user.
4. **Enhanced insights generator** that provides detailed, actionable recommendations based on computed metrics.
5. Updated UI components with clear messages, loading states, and inline validation/error handling.
6. Unit and integration tests covering:
   - Onboarding progression after PDF upload.
   - Calculation correctness for burn multiple and revenue growth.
   - Display and edit of the business summary.
   - Rendering of detailed insights.

### Success Criteria

- Uploading a deck and clicking Continue advances to Step 2 reliably.
- Burn multiple and revenue growth display correct values (or clear messages explaining why not).
- A business summary appears at the top of the Truth Scan page, either from the deck or from an online fetch, and can be edited.
- Insights and recommendations give specific advice derived from the scan results instead of generic statements.
- Users experience clear feedback at every step and can proceed through onboarding without confusion.
